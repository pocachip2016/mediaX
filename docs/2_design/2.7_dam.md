# 2.7 DAM (디지털 에셋 관리)

**역할:** 디자인팀↔편성팀 간 에셋 게이트. 생성·승인·게시·보관 전 주기 관리.

---

## 핵심 기능

### 에셋 등록 및 분류

- 수동 업로드 (Phase 1) / 자동 등록 (Phase 2 인제스트 AX 연동)
- 콘텐츠 ID 기반 자동 태깅: `content_id`, `slot_type`, `device`, `version`
- 에셋 유형 분류: 포스터 / 배너 / 썸네일 / 이벤트 / GUI 소재
- 업로드 시 사이즈 자동 검증 (2.1 규격 기준 불일치 시 경고)

### 워크플로우 (승인 흐름)

```
[등록 요청] → [디자인팀 제작] → [팀장 승인] → [DAM 게시 확정] → [편성팀 픽업]
                                     ↓ 반려
                                [재작업 요청]
```

- 상태값: `draft` / `review` / `approved` / `rejected` / `published` / `archived`
- 승인자 지정: 팀장 또는 위임된 시니어 디자이너
- 반려 시 사유 코멘트 필수 입력

### RBAC 권한 체계

| 역할 | 권한 |
|------|------|
| 디자이너 (정직원) | 등록·수정·제출 |
| 프리랜서 | 등록·수정 (제출 불가, 팀원 확인 필요) |
| 팀장 | 승인·반려·게시 |
| 편성팀 | 조회·다운로드 (수정 불가) |
| 관리자 | 전체 권한 |

### 에셋 검색 및 조회

- 검색 조건: 콘텐츠명, 콘텐츠 ID, 슬롯 유형, 디바이스, 등록일, 상태, 담당자
- Elasticsearch 기반 풀텍스트 검색 + 태그 필터
- 미리보기: 브라우저 내 썸네일 렌더링

### 버전 관리

- 동일 콘텐츠·슬롯 조합에 대해 버전 이력 유지 (v1, v2, …)
- 이전 버전 복원 가능 (팀장 권한)
- 최신 버전만 편성팀에 노출

### 에셋 라이프사이클

```
등록 → 승인 → 게시 → 만료(보관) → 아카이브
```

- 만료 기준: VOD 라이선스 종료일 또는 이벤트 종료일 자동 연동
- 만료 7일 전 알림 → 편성팀 픽업 중단, 디자인팀 아카이브 처리

---

## Phase별 구현 범위

| Phase | 구현 내용 |
|-------|-----------|
| Phase 1 (MVP) | 수동 업로드, 워크플로우, RBAC, 기존 파일 마이그레이션 |
| Phase 2 | 인제스트 AX 스틸컷 자동 등록, AI 생성 이미지 DAM 자동 연동 |
| Phase 3 (검토) | 웹 크롤링 포스터 수집, CP 포털 직접 업로드 연동 |

---

## 연동 포인트

- **편성 AX** → 승인 완료 에셋 URL 전달 (슬롯별 CDN 경로)
- **인제스트 AX** → 스틸컷·포스터 원본 자동 수신 (Phase 2)
- **2.4 배치 시스템** → 대량 생성 결과물 DAM 자동 등록
- **2.5 브랜드 검수** → 검수 통과 여부 태그 반영
