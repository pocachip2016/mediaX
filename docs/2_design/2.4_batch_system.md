# 2.4 대량 생산 배치 시스템

**목적:** 월 500~2,000건의 에셋을 비동기 처리로 대량 생성·검수·DAM 등록까지 자동화.

---

## 배치 처리 흐름

```
[배치 요청 접수]
  → batch_jobs 레코드 생성 (status: queued)
  → content_ids × slot_codes 조합으로 batch_job_items 생성

[Celery 워커 처리 — 병렬]
  각 item별:
    → 템플릿 자동 선택 (2.3)
    → AI 이미지 생성 (2.2)
    → 브랜드 검수 (2.5)
    → DAM draft 등록 (2.7)
    → item status → done / failed

[완료]
  → batch_jobs.status → done
  → 담당자 알림 (이메일 또는 대시보드)
  → 일괄 승인 화면 활성화
```

---

## Celery 큐 구조

| 큐 이름 | 담당 작업 | 워커 수 (MVP) | 우선순위 |
|---------|-----------|--------------|----------|
| `design.high` | 긴급 단건 요청 | 2 | 최상 |
| `design.normal` | 일반 배치 작업 | 4 | 보통 |
| `design.brand_check` | 브랜드 검수 전용 | 2 | 보통 |
| `design.cdn` | CDN 업로드 전용 | 2 | 낮음 |

### 우선순위 정책

- `design_requests.priority = urgent` → `design.high` 큐로 라우팅
- 일반 배치는 요청 시각 순 FIFO
- 야간(22:00~07:00) 대형 배치 자동 스케줄 권장

---

## 에러 처리 및 재시도

| 에러 유형 | 처리 방식 |
|----------|----------|
| 이미지 소스 없음 | AI 생성형으로 fallback, 실패 시 item failed |
| 브랜드 검수 미통과 | 자동 보정 1회 시도 → 그래도 실패 시 수동 확인 요청 |
| AI API 타임아웃 | 30초 후 최대 3회 재시도, 이후 failed |
| CDN 업로드 실패 | 5분 후 재시도 2회, 이후 알림 발송 |

- `failed` 항목만 선택해 **개별 재생성** 가능 (배치 전체 재실행 불필요)
- 재생성 시 다른 템플릿 선택 옵션 제공

---

## 일괄 승인 워크플로우

```
배치 완료 알림
  → 팀장이 배치 결과 대시보드 접근
  → 시안 썸네일 그리드 조회 (슬롯별·콘텐츠별 필터)
  → 일괄 승인 / 선택 승인 / 일괄 반려 처리
  → 승인된 항목 → DAM published + 편성 AX 전달
```

- **개별 수정:** 승인 전 특정 시안만 선택해 웹 에디터(2.6)로 연결
- **부분 승인:** 100건 배치에서 80건 승인 + 20건 재생성 요청 가능

---

## 배치 대시보드

### 표시 항목

| 항목 | 내용 |
|------|------|
| 진행 현황 | 전체/완료/실패 건수, 진행률 프로그레스바 |
| 예상 완료 시각 | 평균 처리속도 기반 추정 |
| 실패 목록 | 에러 유형별 그룹핑, 개별 재시도 버튼 |
| 시안 미리보기 | 슬롯·콘텐츠별 썸네일 그리드 |
| 브랜드 검수 요약 | 통과율, 항목별 미통과 건수 |

### 알림

- 배치 완료 시 담당자 이메일 + 대시보드 알림
- 실패율 20% 초과 시 팀장 긴급 알림
- 24시간 이내 미승인 배치는 팀장에게 리마인더

---

## CDN 자동 배포

- 승인 완료 에셋 → S3(또는 동등 오브젝트 스토리지) 업로드
- CloudFront(또는 동등 CDN) 캐시 무효화 자동 실행
- 슬롯별 CDN URL 패턴: `https://cdn.example.com/design/{content_id}/{slot_code}/v{version}.webp`
- 최신 버전만 활성 URL 유지, 구버전은 30일 후 자동 삭제
