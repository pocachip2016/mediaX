# 2.8 연동 포인트

---

## 수신 (디자인 AX ← 타 모듈)

| 출처 | 전달 데이터 | 용도 | 시점 |
|------|------------|------|------|
| **메타데이터 AX (1.1)** | 장르, 감성 태그, 인물 정보 | 템플릿 자동 선택, AI 프롬프트 생성 | 콘텐츠 등록 시 |
| **카탈로그 AX (1.2)** | 콘텐츠 ID, 제목, CP사, 유형(단편/시리즈) | 에셋 태깅, DAM 자동 분류 | 카탈로그 확정 시 |
| **인제스트 AX** | 스틸컷 원본, 포스터 원본 URL | AI 이미지 생성 소스, DAM 등록 | Phase 2 자동화 |
| **CP 포털** | 디자인 요청 (콘텐츠 ID + 슬롯 유형) | 작업 큐 생성 | 요청 수신 시 (Phase 2) |

## 송신 (디자인 AX → 타 모듈)

| 목적지 | 전달 데이터 | 용도 | 시점 |
|--------|------------|------|------|
| **편성 AX** | 승인 완료 에셋 CDN URL, slot_code | 홈 슬롯 배치·배너 등록 | DAM 게시 확정 시 |
| **배포 모듈 (10)** | 에셋 CDN 경로, 디바이스별 URL | 플랫폼 배포 | 편성 요청 시 |
| **통계 AX** | 에셋 ID, 슬롯 유형, 게시일 | 이미지 성과 측정 | 게시 이벤트 발생 시 |

## API 규격 (FastAPI 기준)

### 에셋 등록 (DAM 수신)
```
POST /api/design/assets
Body: {
  content_id: str,
  slot_code: str,         // 예: tv_poster_lg
  file_url: str,
  source: str,            // manual | ai_generated | ingest_auto
  created_by: str
}
Response: { asset_id, status: "draft" }
```

### 에셋 조회 (편성팀 픽업)
```
GET /api/design/assets?content_id=&slot_code=&status=approved
Response: [{ asset_id, cdn_url, version, approved_at }]
```

### 승인 처리
```
PATCH /api/design/assets/{asset_id}/status
Body: { status: "approved" | "rejected", comment: str }
```

### 배치 생성 요청 (내부)
```
POST /api/design/batch
Body: {
  content_ids: [str],
  slot_codes: [str],
  template_id: str        // optional, 미지정 시 AI 자동 선택
}
Response: { batch_id, queued_count }
```

## 이벤트 흐름 요약

```
[카탈로그 확정]
  → 디자인 AX 작업 큐 생성
  → 배치 시스템 템플릿 선택 + AI 렌더링
  → 브랜드 검수 자동 통과
  → DAM draft 등록
  → 팀장 승인
  → 편성 AX에 CDN URL 전달
  → 편성팀 슬롯 배치
```
