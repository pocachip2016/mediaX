# 03. 콘텐츠 추가 통합 UX

> Step 2 산출물 — 단일/CSV/외부 검색 3가지 입력 방식을 하나의 진입점으로 통합. wireframe ASCII + 상호작용 흐름.

## 1. 설계 원칙

### 1.1 핵심 원칙

1. **단일 진입점**: 콘텐츠 목록 페이지 우상단 `[+ 콘텐츠 추가 ▾]` 버튼 하나로 모든 입력 방식 시작
2. **방식별 학습 비용 최소화**: 3가지 입력 방식이 같은 모달/페이지 내에서 탭으로 분기
3. **공통 UX 패턴**: 입력 → 미리보기 → 확정 (3 단계)
4. **상태 일관성**: 어떤 방식이든 등록 후 `waiting` 상태로 통일
5. **에러 친화적**: 미리보기 단계에서 모든 검증 결과 표시, 부분 등록 허용

### 1.2 진입점 비교 (현재 vs 신규)

```
현재:
  사이드바 > 메타데이터 > 실시간 메타 생성 (단일)
  사이드바 > 메타데이터 > 배치 업로드 (CSV)
  외부 검색에서 가져오기는 ❌ 없음

신규:
  사이드바 > 콘텐츠 관리 > 콘텐츠 목록
                        └ 우상단 [+ 콘텐츠 추가 ▾]
                                  ├ 단일 입력
                                  ├ CSV 배치
                                  └ 외부 검색
```

## 2. Wireframe — 콘텐츠 추가 모달

### 2.1 진입: 콘텐츠 목록 페이지 우상단 액션

```
┌────────────────────────────────────────────────────────────────────────┐
│  📋 콘텐츠 목록                            [+ 콘텐츠 추가 ▾]            │
│                                            ┌─────────────────────┐    │
│                                            │ ➕ 단일 입력          │    │
│                                            │ 📄 CSV 배치 업로드    │    │
│                                            │ 🔍 외부 검색에서      │    │
│                                            └─────────────────────┘    │
└────────────────────────────────────────────────────────────────────────┘
```

- 드롭다운 항목 클릭 → 모달 오픈, 해당 탭 선택된 상태로
- 모달 내에서 탭 전환 가능 (의도 변경 시)
- 모달 크기: **풀스크린 ¾** (양 끝에 살짝 여백, 콘텐츠 목록이 뒤에 보임 — 컨텍스트 유지)

**상호작용**:
- 파싱 후 즉시 정상/누락/오류/중복 4분류
- 오류 행은 기본 건너뛰기 — 강제 등록 옵션은 비권장으로 표시
- `[N건 등록 ✓]`: 1,199건 등록 시작 → ContentBatchJob 생성 → 백그라운드 처리 → toast 알림 + 콘텐츠 목록 이동
- 등록 후 진행 상황은 콘텐츠 목록의 "처리 중" 필터 또는 처리 현황(`/contents/pipeline`) 페이지에서 추적

## 3. 백엔드 API 매핑

| UI 액션 | 백엔드 API | 비고 |
|---------|----------|------|
| 단일 입력 → 등록 | `POST /metadata/contents` | 기존 |
| 단일 입력 → AI 미리 생성 | `POST /metadata/generate` | 기존 |
| 단일 입력 → 외부 매칭 미리보기 | `POST /metadata/contents/{tmp_id}/enrich?preview=true` | **신규 — 등록 전 미리보기** |
| CSV 배치 업로드 → 등록 | `POST /metadata/upload/batch` | 기존 |
| CSV 배치 미리보기 | `POST /metadata/upload/batch?preview=true` | **신규 — dry-run 옵션** |
| 외부 검색 | `GET /metadata/sources/search?q=...&sources=tmdb,kobis` | **신규 — 통합 검색** |
| 외부 검색 → 등록 | `POST /metadata/contents/from_sources` | **신규 — 매칭 결과 기반 생성** |

신규 API 4개 추가 필요. 별도 task 로 백엔드 작업 분리.

| 백그라운드 등록 중 모달 닫기 | toast + 콘텐츠 목록 이동, job_id 추적 가능 |
