# 04. 콘텐츠 상세 화면 — 5개 메타 탭 세부 설계

> Step 3 산출물 — `/programming/contents/[id]` 단일 화면이 흡수해야 할 5개 메타 탭(글자/이미지/영상/외부소스/AI이력) 의 필드·액션·AI 표시 패턴.

기존 분산 화면(`metadata/text`, `metadata/image`, `metadata/image/upload`, `metadata/video`, `metadata/video/qc`, `metadata/staging`, `metadata/queue`, `metadata/text/review`)을 한 콘텐츠 단위로 묶는다. 데이터 모델은 `backend/api/programming/metadata/models/` 그대로 유지.

---

## 1. 페이지 헤더 (5개 탭 공통)

### 1.1 wireframe

```
┌───────────────────────────────────────────────────────────────────────┐
│ ← [목록]   |  기생충 (2019)  · 영화 · CJ ENM           [⋯ 더보기]   │
│                                                                       │
│  상태: [🟧 검수]   품질: ████████░░ 82      ID: #1247                 │
│         review            quality_score                               │
│                                                                       │
│  [✓ 승인]  [✗ 반려]  [✏ 수정]  [↻ AI 재처리]  [🔍 외부 재매칭]      │
│   ───── 컨텍스트 액션 (status 에 따라 동적 노출) ─────                │
└───────────────────────────────────────────────────────────────────────┘
┌───────────────────────────────────────────────────────────────────────┐
│  [글자 ●]  [이미지 ◉]  [영상 ○]  [외부소스 3]  [AI 이력 12]          │
│   completed completed pending   matched n     log entries            │
└───────────────────────────────────────────────────────────────────────┘
```

### 1.2 탭 라벨 + 부속 표식

| 탭 | 라벨 | 우측 표식 | 의미 |
|----|------|---------|------|
| 1 | 글자 | `●` (체크) / `◉` (검수 필요) / `○` (미완) | `text_meta_completed` + AI 신뢰도 |
| 2 | 이미지 | 동일 | `image_meta_completed` |
| 3 | 영상 | 동일 | `video_meta_completed` |
| 4 | 외부소스 | 정수 N | 매칭된 `ExternalMetaSource` 갯수 |
| 5 | AI 이력 | 정수 N | `ContentAIResult` 레코드 갯수 |

→ 운영자가 한눈에 **어떤 탭에 액션이 필요한지** 파악. `◉` 면 그 탭 안에 review 대기 항목 존재.

### 1.3 상단 액션 버튼 — status 매트릭스

| status | 노출 버튼 |
|--------|---------|
| `waiting` | [▶ 즉시 처리] |
| `processing` | (없음 — 진행 중 스피너만) |
| `processing.error` | [↻ 재시도] [✏ 수동 입력 전환] |
| `staging` | [✓ 승인] [✗ 반려] [✏ 수정] |
| `review` | [✓ 승인] [✗ 반려] [✏ 수정] [↻ AI 재처리] [🔍 외부 재매칭] |
| `approved` | [↶ 회수] (관리자만) |
| `rejected` | [↶ 회수] (관리자만) |

→ `02_menu_lifecycle.md` §4.2 수동 전이 표와 1:1 매핑.

### 2.1 데이터 소스

```
└───────────────────────────────────────────────────────────────────────┘
```

### 2.3 핵심 패턴 — "필드 × 3소스 카드"

각 메타 필드는 **CP 원본 / AI 결과 / 외부 매칭** 의 3 카드로 보여준다.

- **헤더 칩**: `[CP] trust 100` / `[AI] score 87` / `[TMDB] match .91`
- **본문**: 잘려진 값 (시놉시스는 길이/줄임표 처리)
- **하단 라디오**: 어떤 소스를 `final_*` 로 채택할지 1개 선택
- **[✏ 직접 작성]** 클릭 시 → 4번째 카드 `[수동]` 추가 + 텍스트 영역 열림

→ 신뢰도 변환 규칙:
- CP: 항상 `trust 100` (원본 신뢰)
- AI: `ContentAIResult.quality_score` (0~100)
- 외부: `ExternalMetaSource.match_confidence` (0~1) → 소수 둘째자리

### 2.4 액션 버튼

| 버튼 | 동작 | 호출 API |
|------|------|---------|
| [✏ 직접 작성] | 수동 입력 카드 열기 | (FE 로컬 상태) |
| [↻ AI 재생성] | 해당 필드만 재처리 | `POST /metadata/contents/{id}/process?fields=synopsis` |
| [→ 채택] | 대안 후보를 final 로 승격 | `PUT /metadata/contents/{id}` 부분 업데이트 |
| [💾 저장 후 메타 완료] | `final_*` 확정 + `text_meta_completed=true` | `PUT /metadata/text/{id}` |
| [💾 임시 저장] | 중간 상태 저장 (completed flag 그대로) | `PUT /metadata/text/{id}` |

### 2.5 검수 모드 (행 상태 = review)

review 상태에선 **누락/저신뢰 필드만 강조**:
- AI 점수 < 70 인 필드 → 노란 테두리
- 빈 final 필드 → 빨간 테두리 + "필수 입력" 라벨
- 정상 필드는 접힌 상태로 표시 (▸ 클릭 펼침)

→ 검수자 시선이 **결정이 필요한 필드** 에만 집중되도록.

## 3. 탭 #2 — 이미지 메타

### 3.1 데이터 소스

`content_images` 테이블, `ImageType` enum (poster/thumbnail/stillcut/banner/logo).


### 3.3 핵심 패턴

- **타입별 그리드** — 한 콘텐츠가 가진 모든 이미지를 `ImageType` 으로 묶어 표시
- **대표 표시** — `is_primary=true` 인 항목에 ⭐ 배지. 대표는 타입당 1개
- **소스 칩** — `ContentImage.source` (cp/tmdb/manual/watcha/kobis) 칩으로 출처 명시
- **검수 체크박스** — 각 이미지의 ◯/✓ 토글. 이 자체로 `image_meta_completed` 계산 (모두 ✓ 면 완료 가능)

### 3.4 액션

| 버튼 | 동작 |
|------|------|
| [⭐] | 대표 이미지 토글 (`is_primary`) — 같은 타입 내 다른 대표 자동 해제 |
| [🗑] | 이미지 삭제 (관리자만) |
| [+ 이미지 업로드 ▾] | 모달 (파일/URL/TMDB) — 단일 콘텐츠에서 외부 가져오기 가능 |
| [✓ 전체 검수 완료] | 모든 이미지가 검수된 상태에서만 활성 → `image_meta_completed=true` |

### 3.5 검수 흐름

기존 `metadata/image/upload` 화면이 **단일 콘텐츠 컨텍스트** 안으로 흡수됨. 일괄 업로드는 [+ 이미지 업로드] 모달의 다중 파일 드롭존으로 지원.

## 4. 탭 #3 — 영상 메타

### 4.1 데이터 소스

### 4.3 핵심 패턴 — 자동 vs 수동 입력 구분

영상 메타는 대부분 **ffprobe 자동 감지** 결과 + 운영자 보정. 표시 방식:

- 자동 감지 값 → 회색 입력란 + `🔧 자동 감지` 칩
- 수동 수정 후 → 흰 배경 + `✏ 수동` 칩
- 미감지 → 빈 입력란 + ⚠ 경고

### 4.4 QC 체크리스트 — `metadata/video/qc` 흡수

기존 별도 QC 화면을 탭 내부 체크리스트로 통합. 항목:

| 체크 | 의미 |
|------|------|
| 영상 재생 가능 | preview_clip_url 재생 성공 |
| 자막 싱크 | 자막 트랙 정상 |
| 등급 워터마크 | 시청등급 워터마크 삽입 확인 |
| 광고 삽입점 | midroll/postroll 마커 정의 |

→ 4/4 통과 + 필수 기술 메타 입력 시 `video_meta_completed=true` 가능.

- **소스별 카드** — `ExternalSourceType` 9종 중 매칭된 것만 표시. 미매칭은 하단 회색 영역
- **신뢰도 시각화** — `match_confidence`:
  - ≥ 0.9: 초록 칩
  - 0.7 ~ 0.89: 주황 칩
  - < 0.7: 빨강 칩 + ⚠ 경고
- **원본 JSON 펼침** — `raw_json` 전체 보기 (디버깅/검증용)

### 5.4 [📋 필드별 가져오기] — 핵심 운영자 액션

클릭 시 모달 열림:

```
┌──────────────────────────────────────────────────────────┐
```

### 5.5 [🔍 외부 재매칭] 흐름

```

    ↓
[재매칭 실행]
```

### 6.3 핵심 패턴

- **`is_final` 토글** — ● = 현재 채택, ○ = 대안. 같은 task 안에서 한 번에 1개만 ●.
- **태스크 6종** — `AITaskType` enum (synopsis/genre/tagging/rating/entity/quality).
- **엔진 비교 막대** — 같은 태스크의 엔진별 점수를 가로 막대로 비교 → 운영자가 "어떤 엔진이 이 콘텐츠에 좋은지" 직관 파악.
- **에러 행** — `error_message` 가 있으면 ❌ 배지 + [에러 보기] (모달로 stack trace).

### 6.4 액션

| 버튼 | 동작 |
|------|------|
| [채택] | `is_final` 을 해당 row 로 옮김 + `final_*` 필드 갱신 | `PUT /metadata/contents/{id}/ai-results/{result_id}/promote` (신규) |

| [보기] | 우측 미리보기 영역 갱신 | (FE) |

### 6.5 운영자 가치

- **디버깅 컨텍스트** — "왜 이 시놉시스가 채택됐지?" → 엔진/점수/원본 JSON 추적
- **엔진 평가** — 콘텐츠 타입(영화/시리즈)별로 어떤 엔진이 정확한지 데이터 축적
- **회수 시 대안 즉시 채택** — approved 회수 후 review 로 돌아왔을 때 즉시 다른 엔진 결과로 교체 가능

## 7. 공통 패턴 — "필드 × 소스" 표시 규약

5개 탭 전반에서 일관되게 적용할 시각 규약:

### 7.1 소스 칩 색

| 소스 | 색 | 의미 |
|------|----|------|
| `CP` | 회색 | CP 원본 (변경 불가, 신뢰 기본) |
| `AI` | 보라 | AI 생성 (점수 동반 필수) |
| `TMDB` | 파랑 | The Movie Database |
| `KOBIS` | 초록 | 영화진흥위원회 |
| `KMDB` | 진초록 | 한국영상자료원 |
| `Watcha`/`Netflix` | 주황 | 비공식 소스 (확인 필요) |
| `수동` | 검정 | 운영자 직접 입력 |

### 7.2 신뢰도 색

| 점수/match | 색 | 표기 |
|-----------|----|------|
| ≥ 90 / ≥ .90 | 초록 | `score 92` / `match .92` |
| 70~89 / .70~.89 | 주황 | `score 82` / `match .82` |
| < 70 / < .70 | 빨강 + ⚠ | `score 64` / `match .64` |

### 7.3 변경 이력 (모든 탭 우측 사이드 패널, 토글)

```
┌──────────────────────────────┐
│ 📜 변경 이력           [×]    │
├──────────────────────────────┤
│ 2026-05-12 14:30  by AI     │
│   synopsis: gemini 채택      │
│ 2026-05-12 14:30  by AI     │
│   genres: TMDB 매칭          │

```

→ `content_ai_results` + 운영자 액션 로그 (감사 테이블) 결합. (별도 백엔드 task — 운영자 액션 로그 테이블 신규)

### 7.4 필드 잠금 (수동 우선)

운영자가 수동 입력한 필드는 **🔒 잠금 표시** → AI 재처리 시 덮어쓰지 않음.

- 잠금 해제: [🔒 클릭] → 재처리 허용
- 백엔드: `ContentMetadata` 에 `locked_fields: JSON` 컬럼 추가 (별도 task)
