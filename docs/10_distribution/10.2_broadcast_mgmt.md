# 10.2 송출 관리

## 개요

CDN에 업로드된 콘텐츠 파일의 퍼지(Purge) 관리, 포털 UI 데이터 전송, DA(Display Ad) 소재 전송을 담당한다. 콘텐츠 업데이트·삭제·교체 시 정확한 데이터가 즉시 반영되도록 보장한다.

---

## 1. 콘텐츠 파일 퍼지 (Purge)

### 개요

CDN에 캐시된 파일(영상, 썸네일, 메타데이터 JSON 등)을 강제로 무효화하여 최신 파일이 즉시 서빙되도록 한다. 콘텐츠 수정, 삭제, 메타 업데이트 후 필수적으로 실행한다.

### 퍼지 발생 트리거

| 트리거 | 퍼지 대상 | 우선순위 |
|--------|----------|---------|
| 콘텐츠 메타 수정 (제목, 등급, 가격 등) | 메타 JSON, 썸네일 | 일반 |
| 썸네일 이미지 교체 | 썸네일 파일 | 일반 |
| 콘텐츠 서비스 중단/삭제 | 영상 파일, 메타 JSON | 긴급 |
| 가격 변경 | 메타 JSON | 일반 |
| 영등위 등급 변경 | 메타 JSON, 등급 이미지 | 긴급 |
| 자막 파일 교체 | 자막 파일 | 일반 |

### 퍼지 처리 워크플로우

```
퍼지 요청 생성 (수동 or 자동 트리거)
    │
    ▼
대상 URL 목록 생성
    ├── 단일 파일 퍼지: 특정 URL 지정
    └── 와일드카드 퍼지: 디렉터리 전체 무효화
    │
    ▼
CDN API 호출 (KT CDN / 외부 CDN)
    │
    ├── 성공 → 퍼지 완료 로그 기록
    └── 실패 → 재시도 (3회) → 실패 알림
    │
    ▼
퍼지 완료 확인 (Cache-Control 응답 헤더 검증)
```

### 퍼지 관리 UI

- 수동 퍼지 요청 폼 (URL 직접 입력 또는 콘텐츠 ID 검색)
- 퍼지 이력 조회 (요청자, 일시, 대상 URL, 결과)
- 퍼지 실패 목록 및 재시도 버튼
- 배치 퍼지 (여러 콘텐츠 일괄 처리)

## 2. 포털 UI 데이터 전송

### 개요

지니TV 포털 및 외부 플랫폼(웹, 앱)의 UI 화면에 표시되는 상품 정보(상품명, 가격, 썸네일 URL, 서비스 기간 등)를 자동으로 동기화한다.

### 전송 데이터 유형

| 데이터 유형 | 전송 주기 | 우선순위 |
|------------|----------|---------|
| 신규 콘텐츠 등록 정보 | 실시간 (등록 즉시) | 높음 |
| 가격 변경 | 실시간 | 높음 |
| 서비스 기간 변경 (홀드백) | 실시간 | 높음 |
| 썸네일/이미지 URL | 퍼지 후 자동 | 보통 |
| 기획전·이벤트 구성 | 편성 일정에 따라 | 보통 |
| 배너 이미지 및 링크 | 편성 일정에 따라 | 보통 |
| 카테고리/섹션 구성 | 수동 변경 시 | 낮음 |

### 전송 인터페이스

- **API 방식:** 포털 시스템에 REST API로 데이터 Push
- **메시지 큐 방식:** Kafka/RabbitMQ 통해 비동기 전송
- **배치 방식:** 일괄 변경사항 야간 배치 동기화

### 전송 현황 모니터링

- 전송 성공/실패 건수 실시간 집계
- 실패 건 재전송 큐 관리
- 포털 반영 지연 시간 추적 (SLA: 실시간 변경 5분 이내)

## 3. DA 소재 전송

### 개요

콘텐츠 프로모션을 위한 DA(Display Advertisement) 소재(배너 이미지, 동영상 광고 등)를 광고 플랫폼 및 외부 매체에 자동 전송한다.

### DA 소재 유형

| 소재 유형 | 사이즈 | 게재 위치 |
|----------|--------|---------|
| 메인 배너 | 1920×600 | 포털 메인 |
| 서브 배너 | 640×240 | 카테고리 페이지 |
| 모바일 배너 | 750×400 | 모바일 앱 상단 |
| 프리롤 광고 | 1280×720 (15/30초) | 영상 재생 전 |
| 스퀘어 배너 | 600×600 | SNS 광고 |

### 전송 프로세스

```
디자인 AX에서 소재 생성 완료
    │
    ▼
브랜드 검수 통과 확인 (2.5_brand_check)
    │
    ▼
게재 일정·타겟 조건 매핑
    │
    ▼
광고 플랫폼 API 전송
    ├── 내부 인벤토리 (포털, 앱)
    └── 외부 매체 (Google, Kakao 등)
    │
    ▼
게재 시작 확인 → 노출 모니터링
```

### 연동 시스템

- **마케팅 AX (`5_marketing`):** 캠페인 일정·타겟 조건 수신
- **디자인 AX (`2_design`):** 완성된 소재 파일 수신
- **6_monitoring:** 게재 오류 알림 수신
