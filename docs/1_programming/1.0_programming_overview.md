# 1.0 편성 기획 AX 개요

**미션:** 지니TV VOD 편성 기획 전 과정에 AI 적용, 데이터 기반 의사결정 체계 구축

## 현황
- **월 신규 VOD:** 1,000~5,000건
- **장르 범위:** 영화/드라마/예능/키즈/애니/교육/성인/스포츠/음악 (전 장르)
- **메타 입력:** 혼재 (CP사마다 다름)
- **현재 환경:** ERP/CMS 등 전용 시스템 사용 중

---

## 모듈 구성 (5개)

| 모듈 | 파일 수 | 설명 |
|------|--------|------|
| 1.1 메타데이터 AI 자동 분류 | 9개 | CP 메타 표준화, AI 5개 엔진, VAMS, 외부 메타 수집 |
| 1.2 VOD 카탈로그 관리 자동화 | 9개 | 등록~종료 라이프사이클, 중복탐지, 라이선스 만료 관리 |
| 1.3 홈 큐레이션 AI | 8개 | 홈 슬롯 자동 운영, 개인화 추천, 트렌드 기반 기획전 |
| 1.4 편성 결재·워크플로우 | 6개 | 승인 플로우 자동화, 이상 편성 감지, 변경 이력 추적 |
| 1.5 CP사 수급 관리 | 6개 | CP 성과 분석, 수급 AI 스코어링, 판권계약(CMT) |

## 모듈별 상세

### 모듈 1.1 — 메타데이터 AI 자동 분류

CP사마다 제각각인 메타 품질을 표준화하는 **최우선 모듈**. 전체 모듈의 공통 기반 데이터 소스 역할.

| 파일 | 내용 |
|------|------|
| `1.1.1_data_pipeline` | 입력 3가지: CP 제공 / 웹 크롤링(TMDB·KMDB) / 인제스트 AX 영상 분석 |
| `1.1.2_ai_engines` | 5개 엔진 병렬 처리: ①장르분류 ②시놉시스생성 ③인물태깅 ④감성태깅 ⑤품질스코어(0~100) |
| `1.1.3_workflow` | 스코어 기반 3단계 라우팅: 90+ 자동등록 / 70~89 담당자 리뷰 / 70미만 반려 |
| `1.1.4_ui_screens` | 담당자 검수 UI 화면 |
| `1.1.5_integration` | 연동 포인트: 인제스트 AX, 카탈로그, 큐레이션, 디자인 AX |
| `1.1.6_vams_overview` | **VAMS** (Video Asset Management): 인제스트 영상에서 키프레임 추출·장면분류·얼굴인식 → 디자인 AX / 외부 배포 연결 |
| `1.1.7_vams_policy_monitoring` | VAMS 추출 규칙 및 서버 모니터링 |
| `1.1.8_external_meta` | 영진위 KOBIS API + TMDB API + OTT 인기도 수집. 메타 병합 우선순위 규칙. 부가기능: Scene Search / AI 자동 등급 고지 / AI 하이라이트 클립 생성 |

**AI 기술 스택:** 멀티모달 LLM + Vision AI + NER(개체명 인식) + 영상/음성 분석

### 모듈 1.2 — VOD 카탈로그 관리 자동화

콘텐츠 등록부터 종료까지 전체 라이프사이클 자동화. 수동 등록 병목·중복·만료 방치 문제 해결.

| 파일 | 내용 |
|------|------|
| `1.2.1_category_tree` | N-depth 카테고리 트리. 단편(Single)/시리즈(Series) 유형 구분, 시즌·에피소드 계층 |
| `1.2.2_pricing` | CP 템플릿 기반 가격 자동 산정 |
| `1.2.3_holdback` | 창구별 서비스 기간 제어 |
| `1.2.4_auto_register` | CP 메타 수신 → AI 분류 → 중복탐지 → 품질확인 → 자동/수동 등록 파이프라인 |
| `1.2.5_duplicate_detect` | 중복 콘텐츠 자동 탐지 |
| `1.2.6_license` | 만료 D-90/30/7/1 단계 알림 → 자동 비공개. AI 갱신 추천 |
| `1.2.7_lifecycle` | 등록 → 게시 → 만료 → 아카이브 전 주기 |
| `1.2.8_db_schema` | DB 스키마 |

### 모듈 1.3 — 홈 큐레이션 AI

홈 화면 배너·테마관·장르 리스트를 AI 자동 추천, 개인화 고도화, 트렌드 기반 자동 기획전 운영.

| 파일 | 내용 |
|------|------|
| `1.3.1_home_slots` | 슬롯 6종: A(메인배너) B(테마관) C(개인화) D(장르) E(랭킹) F(프로모션). 디바이스·시간대별 설정 |
| `1.3.2_banner_ai` | 배너 콘텐츠 AI 자동 선정 |
| `1.3.3_theme_auto` | 트렌드/시즌 기반 기획전 자동 생성 |
| `1.3.4_personalization` | 3단계: 협업 필터링 → 콘텐츠 기반 → 컨텍스트(시간/요일) 개인화 |
| `1.3.5_trend_engine` | 외부 OTT 인기 신호 + 자사 시청 데이터 트렌드 감지 |
| `1.3.6_ab_test` | 슬롯별 A/B 테스트 |
| `1.3.7_db_schema` | DB 스키마 |

### 모듈 1.5 — CP사 수급 관리

CP별 성과 분석, 수급 우선순위 AI 스코어링, 계약 갱신/신규 수급 자동 알림. **CMT(Content Management Tool)** 중심.

| 파일 | 내용 |
|------|------|
| `1.5.1_cp_dashboard` | CP별 성과 대시보드 |
| `1.5.2_supply_scoring` | 과거 성과 + 트렌드 + 자사 시청 패턴 + 경쟁 분석 → 수급 우선순위 0~100점 |
| `1.5.3_contract_mgmt` | 계약 갱신/신규 수급 자동 알림 |
| `1.5.4_db_schema` | DB 스키마 |
| `1.5.5_rights_contract` | **판권계약(CMT):** 독점/비독점/윈도우 계약 유형, RS/MG/Fixed/CPV 정산 기준, 해외환율(한국은행 API), 윈도우 라이선스(극장→PPV→월정액→OTT 창구 순서), 입고예정 콘텐츠 관리 |

## 전체 데이터 흐름

```
CP 메타 수신
  → 1.1 메타 AI 분류 (품질 스코어링)
      → 1.2 카탈로그 자동 등록 (중복탐지·가격·라이선스)
          → 1.4 결재 워크플로우 (승인/반려/에스컬레이션)
              → 1.3 홈 큐레이션 AI (슬롯 자동 배치)

1.1 VAMS      → 2_design (이미지 소스)
1.1.8 외부 OTT 인기도 → 1.5 수급 스코어링 입력
1.5.5 판권계약 → 4_analytics 정산 자동화 연동
```
