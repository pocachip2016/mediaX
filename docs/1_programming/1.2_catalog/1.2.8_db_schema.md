# 1.2.8 DB 스키마 (11개 테이블)

- **categories** — 트리 구조 (id, parent_id, name, depth, sort_order)
- **contents** — 마스터 (id, type[single/series], status, cp_id)
- **series** — 시리즈 (series_id, season_no, content_id)
- **episodes** — 에피소드 (episode_id, series_id, season_no, ep_no)
- **pricing** — 가격 (content_id, quality, purchase_type, price)
- **holdback_policies** — CP별 홀드백 (cp_id, window_no, duration, price_rule)
- **holdback_schedules** — 윈도우 스케줄 (content_id, window_no, start/end_date, price_id)
- **content_categories** — 다대다 매핑
- **price_change_log** — 가격 변경 이력
- **licenses** — 계약 정보
- **lifecycle_events** — 상태 변경 감사 로그
