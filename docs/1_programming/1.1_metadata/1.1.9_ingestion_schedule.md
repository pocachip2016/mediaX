# 1.1.9 수집 스케줄 및 오케스트레이션

## 도구 역할 분리

| 도구 | 역할 | 적용 시점 |
|------|------|----------|
| **Celery Beat** | 정기 배치 스케줄러 (FastAPI 스택 내 통합) | Phase 1 — MVP 기본 |
| **n8n** | 이메일 감지·노코드 워크플로우 PoC | 파일럿 / Phase 2 확장 |
| **n8n → FastAPI Webhook** | n8n 이벤트 감지 → Celery 큐 전달 | Phase 2 이후 병행 운영 시 |

> **Phase 1 기본 원칙:** 별도 서버 없이 Celery Beat 단일 스택으로 운영. n8n은 PoC 검증 후 점진 도입.

## n8n 워크플로우 설계 (Phase 2)

### 워크플로우 1 — CP 메일 자동 처리

```
[IMAP 노드] CP 메일 수신 감지
    │
    ▼
[Claude API 노드] 본문에서 엔티티 추출
    (제목 / 제작연도 / CP사명 / 수량)
    │
    ▼
[HTTP Request 노드] FastAPI Webhook 호출
    POST /api/ingest/cp-email-received
    │
    ▼
[Celery] process_cp_email 태스크 큐 등록
    │
    ▼
외부 메타 조회 병렬 실행 → DB 저장 (Waiting 상태)
```

### 워크플로우 2 — 신작 외부 메타 주기 수집

```
[Cron 노드] 매일 03:00
    │
    ▼
[KOBIS API 노드] 전일 신규 등록 영화 조회
    │
    ▼
[HTTP Request 노드] FastAPI Webhook
    POST /api/programming/metadata/sync-kobis
    │
    ▼
[Celery] 매핑·병합 배치 처리
```

- [ ] Celery Beat 기반 수집 스케줄러 설정
