# 1.1.1 데이터 입수 파이프라인

## 입력 소스 3가지

| 소스 | 내용 | 수집 방식 |
|------|------|----------|
| **CP사 제공 메타** | 제목, 장르, 출연진, 시놉시스, 포스터 (포맷 비표준) | 이메일 자동 감지 / FTP / API |
| **웹 크롤링 보강** | TMDB / KMDB / 영진위 KOBIS / 나무위키 누락 메타 | 외부 API + 크롤러 |
| **영상 원본 연동** | 인제스트 AX 수신 완료 트리거 → 영상/음성 분석 | 이벤트 기반 (Celery) |

---

## 메타 표준화 레이어

CP마다 다른 포맷을 수신 즉시 통합 스키마(EBUCore 기반)로 정규화.

- 장르 코드 매핑 테이블 — CP별 장르명 → 지니TV 표준 코드
- 인물명 통일 사전 — 동명이인 처리, 한/영/한자 통합
- AI가 매핑 이력 지속 학습 → 정확도 자동 향상

---

## CP 메일 자동 감지 파이프라인

CP사가 수급 예정 콘텐츠를 이메일로 통보하는 경우 자동 처리.

```
CP 발송 이메일 수신
    │
    ▼
이메일 폴링 (Celery Beat — 5분 주기 or n8n IMAP 노드)
    │
    ▼
Claude API — 본문에서 제목·제작연도·CP사·수량 추출 (Entity Extraction)
    │
    ▼
외부 메타 자동 조회 병렬 실행 (TMDB / KOBIS / OTT)
    │
    ▼
DB 저장 — 상태: Waiting (수신 대기)
    │
    ▼
담당자 알림 (수신 예정 콘텐츠 목록 리포트)
```

### 이메일 처리 도구 선택

| 옵션 | 특징 | 권장 시점 |
|------|------|----------|
| **Celery Beat + imaplib** | FastAPI 스택 내 통합, 별도 서버 불필요 | Phase 1 (MVP) |
| **n8n IMAP 노드** | 노코드 워크플로우, 빠른 프로토타이핑 | 파일럿/PoC |
| **n8n → Celery 연동** | n8n 감지 → FastAPI Webhook → Celery 큐 | Phase 2 이후 확장 시 |

## 정기 수집 스케줄

> 상세 스케줄 설계는 `1.1.9_ingestion_schedule.md` 참조

| 수집 대상 | 주기 | 방식 |
|----------|------|------|
| 신규 콘텐츠 외부 메타 | 실시간 (이벤트 기반) | 인제스트 완료 트리거 |
| CP 메일 폴링 | 5분 | Celery Beat |
| 영진위 KOBIS 업데이트 | 매일 새벽 3시 | Celery Beat |
| TMDB 신규/에피소드 | 매주 월요일 새벽 2시 | Celery Beat |
| OTT 인기도 데이터 | 매일 새벽 4시 | Celery Beat |
| 메타 품질 재평가 | 매일 새벽 1시 | Celery Beat |

## 단편 vs 시리즈 수집 전략

| 구분 | 수집 전략 | 비고 |
|------|----------|------|
| **단편 (Movie)** | 1회성 전체 메타 수집 (줄거리·캐스팅·포스터) | EBUCore 단일 레코드 |
| **시리즈 (Series)** | Parent-Child 구조: 시즌 메인 메타 먼저 수집 → 에피소드 수에 맞춰 반복(Loop) 수집 | 에피소드별 부제·썸네일 별도 수집 |
