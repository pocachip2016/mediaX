## 2. 가공/배포 통계 현황 관리

### 실시간 대시보드 지표

| 지표 | 집계 주기 | 알림 임계값 |
|------|----------|-----------|
| 일일 처리 콘텐츠 수 | 1시간 | — |
| 썸네일 추출 성공률 | 실시간 | 95% 미만 시 경고 |
| AI 태깅 완료율 | 실시간 | 90% 미만 시 경고 |
| 외부 배포 성공률 | 실시간 | 98% 미만 시 경고 |
| 대기 큐 적체 건수 | 실시간 | 100건 초과 시 경고 |

### 기간별 통계 리포트

- 일간 처리량 추이 (최근 30일)

- 장르별 AI 태깅 정확도 분포
- 외부 플랫폼별 배포 성공률

## 3. 썸네일 추출 서버 및 배치 관리

### 서버 구성

| 서버 역할 | 스펙 | 대수 |
|----------|------|------|
| 썸네일 추출 워커 | CPU 집약형 | 기본 2대 (오토스케일) |
| AI 분석 워커 | GPU or 외부 API | 외부 API 사용 (AWS Rekognition 등) |
| 큐 매니저 | Redis / Celery | 1대 |
| 결과 저장 | CDN + PostgreSQL | 공용 인프라 |

### 배치 작업 구성

| 배치 작업 | 주기 | 설명 |
|----------|------|------|
| 신규 콘텐츠 자동 처리 | 실시간 (이벤트 기반) | 인제스트 완료 후 즉시 |
| 미처리 콘텐츠 재시도 | 매 1시간 | 실패 건 자동 재처리 |
| 썸네일 품질 재평가 | 매일 새벽 2시 | 기준 변경 시 전체 재평가 |
| 외부 배포 동기화 | 매 4시간 | 외부 플랫폼 배포 상태 확인 |
| 삭제 정책 적용 | 매일 새벽 3시 | 만료 이미지 자동 삭제 |
| 통계 집계 | 매일 새벽 1시 | 일간 리포트 생성 |

### 서버 상태 모니터링

- 워커 프로세스 생존 확인 (Heartbeat 30초)
- 큐 적체 건수 실시간 알림
- 오토스케일링 트리거 (큐 적체 50건 이상 시 워커 추가)
- 처리 실패율 급증 시 PagerDuty 알림
