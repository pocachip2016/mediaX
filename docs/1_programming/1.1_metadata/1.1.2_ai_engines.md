# 1.1.2 AI 처리 엔진 (5개 서브 엔진)

## LLM / AI 스택 선택지

| 용도 | 프로덕션 (기본) | 로컬 대안 | 비고 |
|------|--------------|---------|------|
| 텍스트 분류·생성 | Claude API (Sonnet) | Ollama / Llama 3 | 비용·응답속도 트레이드오프 |
| 이미지·영상 분석 | Claude Vision / GPT-4o Vision | — | 외부 API 필수 |
| 얼굴 인식 | AWS Rekognition | Google Cloud Vision | |
| OCR | Naver Clova OCR | Google Vision OCR | |
| 이미지 임베딩 | OpenAI CLIP | open-clip (로컬) | 감성 태그용 |

> **Ollama 적용 조건:** 응답 지연 허용 가능한 배치 처리에 한정. 실시간 UI 자동완성은 Claude API 사용.

---

## 엔진 A. 장르/카테고리/태그 자동 분류

- 시놉시스 텍스트 → NLP 분류 (1차 장르 + 2차 세부 장르 + 무드 태그)
- 포스터/썸네일 → Vision 분석으로 분위기·톤 보조 판단
- 기존 분류 이력 학습 → 지니TV 자체 분류 체계 맞춤 최적화

---

## 엔진 B. 시놉시스·줄거리 자동 생성

- CP 제공 시놉시스 품질 판정 (길이 / 완성도 / 중복)
- 부실 시 웹 크롤링 + 영상 자막 분석으로 자동 재생성
- 스포일러 등급 조절: 짧은 요약 / 상세 줄거리 2단계 출력

---

## 엔진 C. 인물 정보 자동 태깅

- 크레딧 OCR + 자막 분석 → 출연진/감독/작가 추출
- 얼굴 인식 (인제스트 AX 연동) → 인물 마스터 DB 매칭
- 신규 인물 자동 등록 제안 → 담당자 승인 후 DB 반영
- Whisper STT로 음성 → 텍스트 변환 → 대화 기반 인물 추출 보조

---

## 엔진 D. 감성·분위기 태깅

### 텍스트/영상 기반
- 영상 색감 / 음악 / 대사 톤 종합 분석
- 태그 예시: `#따뜻한` `#긴장감` `#가족과함께` `#심야감성` `#액션몰입`

### CLIP 이미지 임베딩 기반 (신규)
- 포스터·스틸컷을 CLIP(OpenAI) 또는 open-clip으로 벡터화
- 사전 정의된 감성 태그 후보군과 코사인 유사도 비교 → 상위 태그 자동 부여
- 텍스트 분석 결과와 앙상블하여 최종 태그 신뢰도 산정
- 로컬 대안: `open-clip-torch` (GPU 없이 CPU 배치 처리 가능)

---

## 엔진 E. 메타 품질 스코어링

- 0~100점 자동 산정
- 기준: 필수 필드 충족률 / 분류 신뢰도 / 이미지 해상도 / 시놉시스 품질
- 판정 결과:

| 점수 | 처리 |
|------|------|
| 90점 이상 | 자동 등록 (사후 샘플 검수) |
| 70~89점 | 담당자 리뷰 후 등록 |
| 70점 미만 | AI 보강 제안 + 담당자 필수 검수 큐 등록 |
