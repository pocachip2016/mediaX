## 세션 분할

### 세션 1 — 화면 4: 메타 품질 리포트
범위:

**백엔드:**
- `service.py`에 함수 추가:
  - `get_quality_report(days: int)` — CP별 품질 트렌드 (일별 평균 점수)
  - `get_field_missing_rate()` — 필드별 누락률 (히트맵 데이터)
- `router.py`에 `GET /metadata/reports/quality` 엔드포인트
- `schemas.py`에 `QualityReportResponse` 스키마

**프론트:**
- `app/(main)/programming/metadata/reports/quality/page.tsx` 신규
  - 최근 90일 CP별 라인 차트 (recharts)
  - 필드별 누락률 히트맵 (테이블 기반 색상 시각화)
  - PDF 내보내기는 Phase 2 (현재 스텁 버튼)
- `lib/api.ts`에 `reportsApi.getQuality(days)` 추가

완료 기준: `python3 -c "from api.programming.metadata.service import get_quality_report; print('OK')"`

### 세션 2 — 화면 5: 분류 체계 관리
범위:

**백엔드:**
- `service.py`에 함수 추가:
  - `list_genre_codes()` / `upsert_genre_code()` / `delete_genre_code()`
  - `list_tag_codes()` / `upsert_tag_code()`
- `router.py`에 CRUD 엔드포인트 추가 (장르·태그)
- `schemas.py`에 `GenreCodeUpsert`, `TagCodeUpsert` 스키마

**프론트:**
- `app/(main)/programming/metadata/taxonomy/page.tsx` 신규
  - 탭 구조: 장르 코드 | 태그 사전
  - 장르: 계층 표시 (들여쓰기 트리) + 인라인 편집
  - 태그: 테이블 + 추가/삭제
  - 드래그앤드롭은 Phase 2 (현재 순서 변경 불가)
- `lib/api.ts`에 `taxonomyApi.*` 추가

완료 기준: `python3 -c "from api.programming.metadata.service import list_genre_codes; print('OK')"`
