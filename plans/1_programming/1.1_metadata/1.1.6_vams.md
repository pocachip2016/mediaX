## 의존성
- 선행: 1.1_metadata 기반 구현 완료 (Content, ContentImage 모델 존재)
- 선행: 1.1.9_schedule 세션 1 완료 (Celery 태스크 패턴 확립 후)
- 외부 API: TwelveLabs (Phase 3 영상 분석), AWS Rekognition (Phase 3 얼굴 인식)

## 아키텍처 결정

**VAMS = 별도 모듈** → `api/programming/vams/`
- `api/programming/vams/models/`, `service.py`, `router.py` 독립 패키지
- Content/ContentImage는 `api/programming/metadata/models/`에서 import해서 참조
- `main.py`에 `vams_router` 별도 등록

## 신규 모델 (Phase 1)

| 모델 | 테이블 | 설명 |
|------|--------|------|
| `Keyframe` | `content_keyframes` | 추출된 키프레임 (timestamp, image_url, quality_score) |
| `SceneSegment` | `content_scenes` | 장면 구간 (start_time, end_time, scene_type, tags) |
| `VamsJob` | `vams_jobs` | 처리 작업 이력 (status, started_at, completed_at, error) |

## 세션 분할

### 세션 1 — 모델 + 마이그레이션
범위:
- `api/programming/vams/models/vams.py` 신규 (Keyframe, SceneSegment, VamsJob)
- `alembic/versions/XXXX_vams_tables.py` 마이그레이션
- `schemas.py`에 VAMS 스키마 추가
- 완료 기준: `python3 -c "from api.programming.vams.models.vams import Keyframe; print('OK')"`

### 세션 2 — 키프레임 추출 서비스
범위:
- `api/programming/vams/extractor.py` 신규
  - `extract_keyframes(content_id, video_path)` — ffmpeg subprocess 사용
  - 추출 규칙 적용 (1.1.7 기준: 5~95% 구간, 30초 간격, 최대 200장)
  - 품질 점수 산정 (blur 감지: OpenCV Laplacian variance)
  - ContentImage 테이블에 저장
- `router.py`에 `POST /metadata/{id}/vams/extract` 엔드포인트
- 완료 기준: `python3 -c "from api.programming.vams.extractor import extract_keyframes; print('OK')"`

### 세션 3 — Celery 태스크 + Beat 등록
범위:
- `workers/tasks/vams.py` 신규
  - `run_vams_extraction(content_id)` — 인제스트 완료 트리거 수신
  - `reeval_thumbnail_quality()` — Beat 매일 02:00
  - `apply_vams_deletion_policy()` — Beat 매일 03:30
- `celery_app.py` Beat 스케줄 추가
- 완료 기준: `python3 -c "from workers.tasks.vams import run_vams_extraction; print('OK')"`

### 세션 4 — VAMS 모니터링 UI
범위:
- **백엔드:** `GET /metadata/vams/stats` — 처리 현황 통계 (성공률, 대기 큐, 평균 처리 시간)
- **프론트:** `app/(main)/programming/metadata/vams/page.tsx` 신규
  - 처리 현황 카드 (일일 처리 수, 썸네일 추출 성공률, AI 태깅 완료율)
  - VamsJob 목록 테이블 (상태, 처리 시간)
  - 콘텐츠별 키프레임 그리드 뷰
- 완료 기준: 브라우저 렌더링 + API 연동 확인

### 세션 5 — Phase 3 (조건부, TwelveLabs/Rekognition)
범위: 외부 API 계약 후 착수
- TwelveLabs API → 영상 장면 분석·태깅
- AWS Rekognition → 얼굴 인식 → PersonMaster 매핑
- 인물별 장면 목록 (`PersonMaster ↔ SceneSegment` 매핑 테이블)
