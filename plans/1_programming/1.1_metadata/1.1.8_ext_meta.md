## 세션 분할

### 세션 1 — 병합 로직 확인 및 보완
범위:
- `ai_engine.py` 또는 `service.py`의 KOBIS/TMDB 조회 함수 Grep으로 확인
- 병합 로직이 없거나 불완전하면: `merge_external_meta(content_id)` 함수 구현
  - 위 우선순위 테이블을 코드로 표현
  - 충돌 항목: `ExternalMetaSource.confidence < 0.8` → Content 상태를 `review`로 전환
- `enrich_content_metadata` Celery 태스크에 병합 로직 연결

완료 기준: `python3 -c "from api.programming.metadata.service import merge_external_meta; print('OK')"`

### 세션 2 — OTT 무료 소스 수집 + KOBIS 매핑 신뢰도
범위:
- `sync_ott_popularity` 태스크 실구현 (무료 소스 기반)
- DB에 `ott_popularity_rank` 컬럼 추가 (마이그레이션)
- KOBIS 매핑 신뢰도 점수: 80점 이상 자동 반영, 미만 시 `review` 상태
- `ExternalMetaSource.confidence` 필드 활용 또는 추가

완료 기준: alembic 마이그레이션 + import 확인

## OTT 무료 소스 전략

| 소스 | 방식 | 비고 |
|------|------|------|
| Netflix TOP10 | `top10.netflix.com/data/all-weeks-*.tsv` 공식 CSV 다운로드 | 주간 갱신, 무료·공개 |
| TMDB popularity | TMDB API `discover/movie?sort_by=popularity.desc` | 이미 API 키 있음, 무료 |
| JustWatch | `apis.justwatch.com` 비공식 API (파라미터 노출됨) | 불안정, 폴백용 |
| 왓챠/웨이브/티빙 | HTML 파싱 (공개 차트 페이지) | ToS 회색지대, 요청 속도 제한 필수 |

> **Phase 1 채택:** Netflix TSV + TMDB popularity (안정적 무료)  
> **Phase 2 검토:** JustWatch 또는 국내 OTT HTML 파싱
